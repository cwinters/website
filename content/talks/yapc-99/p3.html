How does it work?

  • The DBI defines an interface and provides methods for the drivers to implement.
  • It also takes care of metadata, error handling and other housekeeping.
  • The Database Driver (DBD) takes care of database-specific implementations, linking to the proper client libraries (where necessary), and conforming to the DBI specifications.
  • Want to write your own? See perldoc DBI::DBD
  • Some existing drivers:
    DBD::Oracle
    DBD::Sybase
    DBD::mysql
    DBD::Informix
    DBD::ODBC
    DBD::CSV
    DBD::XBase
    ...
