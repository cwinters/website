Packaging system

What is a package?

It's just a tarball!

Discrete bundle of functionality (definitions left to the developer)

Most things in OI come in packages

Packages can be upgraded separately from OI

What's in a package?

Config files (action table, SPOPS definitions)

Perl classes (SPOPS objects, content handlers, whatever else you need)

SQL structures (semi-independent format)

Initial data to load

Templates associated with package

Static HTML pages and images

Standalone scripts (run from CLI, cron, etc.)

Documentation! Changelog!
