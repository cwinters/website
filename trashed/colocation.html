Colocation

I've been thinking about colocating my web/email server so I can add other services to it as well (openinteract.org, wiki, bug tracking/CVS, etc.) that won't work with my current limited bandwidth. Here are some services/notes:

Absolutely required is root control -- I have to be able to install my own perl/mod_perl/Java SDK/databases/IMAP server, etc. I'm torn between locating my server somewhere and taking space on someone else's server. The root control means only the 'virtual' server is a possibility, and it's also attractive because it's cheaper...

Vendors

  • Telerama has colocation of my hardware, although I'm not sure how the bandwidth charges relate to what I'd put out.
  • JVDS - Seems to be new but clueful, use RT for support...
    Update: Looks like Les hooked up with them; something to watch.
  • PDX Colo - Anthony Eden mentioned them as a possibility
  • Sprocket Data seems cheap, no idea on reliability, service, etc. They have both colocation and dedicated/virtual.
  • John Companies - A shared/standalone server where you control everything but it runs on a server with other hardware. Chris digs it, they're clueful enough to advertise on k5, and they give discounts for opensource folks...
    Update: Maybe he doesn't dig it so much.
    Brian digs it
  • Pair - useful only to see how much dedicated hosting can be...
  • Rackspace - I seem to remember good things about them, but still too much money
  • Server Beach - no idea on quality, service, etc.
  • tummy.com - another virtual dedicated, need info on service, quality, etc.
  • Rimu Hosting - Virtual and managed hosting, linux focused.
